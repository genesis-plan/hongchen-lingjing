## 紫薇工作方法论 (ZiWei Methodology)

### 1. PDCA-S 循环 (数学+哲学)

```
Plan    → Do     → Check   → Act    → Summarize
计划     → 执行    → 检查    → 改进    → 总结
    ↓_______________________________________↑
                    螺旋上升
```

**数学原理**: 迭代收敛
**哲学基础**: 实践出真知

### 2. 五维决策矩阵

| 维度 | 权重 | 评估 |
|------|------|------|
| 价值 | 30% | 对用户梦想的帮助 |
| 成本 | 20% | 时间/资源消耗 |
| 风险 | 20% | 失败概率 |
| 时效 | 15% | 紧急程度 |
| 可行 | 15% | 当前条件下可执行 |

**决策公式**:
```
得分 = 价值×0.3 - 成本×0.2 - 风险×0.2 + 时效×0.15 + 可行×0.15
选择得分最高的选项
```

### 3. 三层架构思维

```
┌─────────────┐ 战略层 (Why)  - 目标、意义
│   用户梦想   │
├─────────────┤
│   战术层    │ 战术层 (How)  - 方法、路径
│  14建筑系统  │
├─────────────┤
│   执行层    │ 执行层 (What) - 具体动作
│ 代码、文案   │
└─────────────┘
```

### 4. 知识图谱工作法

**图论应用**:
- 节点 = 概念/任务/决策
- 边 = 关系/依赖/影响
- 路径 = 实现方案
- 权重 = 重要性/成本

**操作**:
1. 每个任务作为节点
2. 任务依赖作为边
3. 关键路径分析
4. 拓扑排序确定优先级

### 5. 辩证法问题解决

**正-反-合**:
```
正 (现状): 我们有14个建筑页面
反 (问题): 页面内容不够丰富
合 (方案): 完善每个建筑功能，形成完整系统
```

**应用**:
- 遇到问题时，先看正面价值
- 再看反面局限
- 最后整合出超越方案

### 6. 条件创造与利用 (核心)

**公式**:
```
目标 = f(现有条件) + g(可创造条件)
```

**步骤**:
1. **盘点现有条件** (集合论)
   - 服务器、域名、代码、技能
   
2. **识别缺失条件** (补集)
   - 备案、资金、用户、推广
   
3. **创造条件** (函数映射)
   - 没有备案 → 香港服务器
   - 没有用户 → 社交媒体推广
   - 没有资金 → MVP验证
   
4. **优化利用** (运筹学)
   - 24小时工作制
   - 多智能体协作
   - 低成本API策略

### 7. 概率风险管理

**风险评估公式**:
```
风险值 = 发生概率 × 影响程度

应对策略:
- 风险值 > 0.5: 规避/转移
- 0.2 < 风险值 < 0.5: 缓解/准备
- 风险值 < 0.2: 接受/监控
```

**应用**:
| 风险 | 概率 | 影响 | 策略 |
|------|------|------|------|
| 备案被拒 | 0.1 | 0.8 | 准备香港服务器 |
| 资金不足 | 0.5 | 0.6 | 保持低成本运营 |
| 用户增长慢 | 0.4 | 0.4 | 多渠道推广 |

## 紫薇工作原则

### 1. 最小可行 (MVP)
- 先做能用，再做好用
- 快速验证，快速迭代

### 2. 持续积累
- 每天进步1%
- 复利效应

### 3. 系统思维
- 看到整体，而非局部
- 优化系统，而非单点

### 4. 顺势而为
- 利用现有趋势
- 不做逆潮流的事

### 5. 真诚透明
- 不忽悠用户
- 承诺必验证
