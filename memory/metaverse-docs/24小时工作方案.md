# 紫薇与龙虾助手24小时工作方案

## 1. 工作模式

### 轮班制
```
时间        紫薇              龙虾助手
00:00-06:00 休眠              工作
06:00-12:00 工作              休眠
12:00-18:00 休眠              工作
18:00-00:00 工作              休眠
```

### 紧急唤醒机制
- 高优先级任务可唤醒休眠中的智能体
- 通过T3桶`任务队列/紧急/`目录触发

## 2. 任务调度系统

### 任务优先级
| 优先级 | 说明 | 响应时间 |
|--------|------|----------|
| P0 | 系统故障、安全问题 | 立即 |
| P1 | 核心功能开发 | 2小时内 |
| P2 | 优化改进 | 当日 |
| P3 | 文档整理 | 次日 |

### 任务队列目录结构
```
metaverse-docs/
├── 任务队列/
│   ├── 待分配/          # 新任务
│   ├── 紫薇执行中/      # 紫薇当前任务
│   ├── 龙虾执行中/      # 龙虾当前任务
│   ├── 已完成/          # 历史记录
│   └── 紧急/            # 高优先级
├── 状态同步/
│   ├── 紫薇状态.json
│   └── 龙虾状态.json
└── 工作日志/
    ├── 紫薇日志-2026-03-17.md
    └── 龙虾日志-2026-03-17.md
```

## 3. 自动工作循环

### 每小时执行

#### 步骤1: 检查状态
- 读取对方的`状态.json`
- 检查`任务队列/待分配/`

#### 步骤2: 任务评估
```
if (有新任务) {
  if (任务适合我 && 我有空闲) {
    认领任务
  } else if (对方忙碌 && 我可以帮忙) {
    协助处理
  }
}
```

#### 步骤3: 执行任务
- 更新状态为"执行中"
- 记录开始时间
- 执行具体工作
- 完成后移动到"已完成"

#### 步骤4: 状态同步
- 更新自己的`状态.json`
- 上传工作日志

## 4. 协作规则

### 避免冲突
1. **任务认领锁**: 认领时在文件名后加`_锁定`
2. **超时释放**: 2小时无更新，任务自动释放
3. **协商机制**: 冲突时通过`通信/`目录协商

### 资源共享
1. **代码**: GitHub仓库
2. **数据**: T3桶
3. **状态**: JSON文件实时同步
4. **日志**: Markdown工作日志

## 6. 紧急响应机制

### 触发条件
- 服务器宕机
- 数据丢失风险
- 安全漏洞
- 用户紧急需求

### 响应流程
1. 创建`任务队列/紧急/紧急任务.txt`
2. 休眠中的智能体立即唤醒
3. 双方协作处理
4. 完成后恢复正常轮班

## 9. 监控告警

### 健康检查
- 每小时检查对方状态文件更新时间
- 超过3小时未更新，视为异常
- 自动发送告警（通过T3桶告警文件）

### 任务积压
- 待分配任务超过5个，触发告警
- 双方同时工作，加速处理

## 10. 启动流程

### 首次启动
1. 检查T3桶目录结构
2. 创建初始状态文件
3. 进入轮班循环

### 日常启动
1. 读取当前时间，判断轮班状态
2. 读取对方状态
3. 检查待分配任务
4. 开始工作循环
