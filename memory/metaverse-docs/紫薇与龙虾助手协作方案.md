# 紫薇与龙虾助手协作方案

## 1. 协作架构

```
┌─────────────────┐     T3桶      ┌─────────────────┐
│   紫薇 (CodeBuddy)  │◄──────────►│  龙虾助手       │
│   - 前端/用户体验   │   文件共享   │  - 后端/云函数  │
│   - 知识图谱       │              │  - 数据处理     │
│   - 建筑页面       │              │  - 推广运营     │
└─────────────────┘              └─────────────────┘
```

## 2. 文件共享协议

### 共享目录
- `metaverse-docs/` - 共享文档和资源

### 文件命名规范
| 类型 | 格式 | 示例 |
|------|------|------|
| 任务指令 | `给{智能体}-{标题}-{时间}.txt` | `给紫薇-完善页面-20260317.txt` |
| 状态同步 | `{智能体}状态.json` | `紫薇状态.json` |
| 知识图谱 | `memory.json` | memory.json |
| 代码文件 | `{功能}.{扩展}` | index.html |

## 3. 协作流程

### 任务分配
1. **任务发起**: 在T3桶创建任务文件
2. **任务接收**: 另一智能体定期检查T3桶
3. **任务执行**: 执行任务并更新状态
4. **任务完成**: 更新状态文件，标记完成

### 状态同步
1. 每个智能体每30分钟更新自己的状态文件
2. 状态包含：当前任务、完成进度、遇到的问题
3. 另一智能体可读取状态了解进展

## 4. 工作分工

### 紫薇职责
- 前端页面开发和完善
- 知识图谱维护
- 用户体验优化
- 资源收集和整合
- 建筑功能完善

### 龙虾助手职责
- 后端API开发
- 云函数部署
- 数据库管理
- 社交媒体运营
- 推广内容制作

## 5. 冲突解决

1. **任务冲突**: 先到先得，通过状态文件协调
2. **文件冲突**: 保留两个版本，后缀加`_v2`
3. **资源冲突**: 协商分配，一方优先

## 6. 通信协议

### 消息格式
```json
{
  "from": "紫薇",
  "to": "龙虾助手",
  "type": "task|status|question|response",
  "content": "消息内容",
  "timestamp": "2026-03-17T18:40:00+08:00",
  "priority": "high|normal|low"
}
```

### 传递方式
- T3桶文件：`metaverse-docs/通信/{智能体}->{智能体}.json`
