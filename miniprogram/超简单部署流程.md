# 红尘灵境小程序 - 超简单部署流程

## 📍 项目位置
`C:\Users\Administrator\Desktop\红尘灵境小程序`

---

## 🎯 5步完成部署（全程不超过30分钟）

### 第1步：下载并打开工具（5分钟）

1. 访问这个网址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载"微信开发者工具"（Windows版本）
3. 安装并打开工具
4. 用微信扫码登录

### 第2步：导入项目（2分钟）

1. 在工具里点击左上角 "+" 号
2. 选择"导入项目"
3. 填写下面这些：
   - **项目名称**：红尘灵境
   - **项目目录**：点击"选择"，选这个文件夹 → `C:\Users\Administrator\Desktop\红尘灵境小程序`
   - **AppID**：选"测试号"（或者填入你的正式AppID）
4. 点击"导入"

### 第3步：配置云开发（3分钟）

1. 在工具顶部菜单，点击"云开发"按钮
2. 首次使用会提示开通，点击"开通"
3. 选"按量付费"（有免费额度）
4. 创建或选择环境：
   - **环境名称**：hclj-production
   - **环境ID**：`hclj-8g46g9fd06e2a760`
5. 点击"创建"

### 第4步：上传云函数（10分钟）

**重要：这些云函数必须一个一个上传**

1. 在工具左侧找到 `cloudfunctions` 文件夹
2. 展开 `cloudfunctions`，会看到很多文件夹
3. 对每个文件夹右键 → 选择"上传并部署：云端安装依赖"
4. 需要上传的19个云函数（按顺序上传）：

**基础功能（7个）：**
- `login` 
- `getSkills`
- `submitWish`
- `saveProgress`
- `saveExamResult`
- `saveCharacter`
- `getUserStats`

**扩展功能（12个）：**
- `createSkill`
- `askAI`
- `getLetters`
- `sendLetter`
- `markLetterRead`
- `getPosts`
- `createPost`
- `getActivities`
- `joinActivity`
- `sendFriendRequest`
- `getFriendRequests`
- `acceptFriendRequest`

5. 每个上传大约10-30秒，上传完一个再上传下一个
6. 上传完成后，云函数文件夹图标会变成绿色的云朵图标

### 第5步：创建数据库（10分钟）

1. 在工具顶部菜单，点击"云开发"按钮
2. 左侧菜单点击"数据库"
3. 点击"添加集合"按钮
4. 输入集合名称，创建这15个集合：

**核心数据（5个）：**
- `users` 
- `skills`
- `wishes`
- `honors`
- `examResults`

**扩展数据（10个）：**
- `userCharacters`
- `letters`
- `posts`
- `activities`
- `friendRequests`
- `friends`
- `marketItems`
- `transactions`
- `challenges`
- `challengeRecords`

5. 创建完所有集合后，点击"权限设置"
6. 选择"所有用户可读，仅创建者可写"
7. 点击确定

1. 在工具顶部，点击"编译"按钮
2. 模拟器会显示小程序界面
3. 测试基本功能：
   - 点击"进入灵境"
   - 点击底部菜单切换页面
   - 测试各个殿的功能

**如果看到小程序界面，说明部署成功！** 🎉

## 📱 正式发布（可选，需要正式AppID）

### 如果你有正式AppID：

1. 在工具中，打开 `project.config.json` 文件
2. 找到 `"appid"` 这一行
3. 把 `"appid": ""` 改成 `"appid": "你的正式AppID"`
4. 点击"上传"按钮
5. 填写版本号：`1.0.0`
6. 填写备注：`红尘灵境正式版`
7. 登录微信后台提交审核
8. 审核通过后发布上线

### 如果没有正式AppID：

测试号可以正常使用所有功能，但无法发布上线供其他人使用。
如果要发布上线，需要先注册微信小程序账号。

## ❓ 遇到问题怎么办？

### 问题1：云函数上传失败
**解决方法**：
- 检查网络连接
- 重新右键云函数文件夹
- 再次选择"上传并部署：云端安装依赖"

### 问题2：数据库集合创建失败
**解决方法**：
- 确保云开发环境已经开通
- 检查集合名称是否正确（不要有空格）
- 删除失败的集合，重新创建

### 问题3：点击"云开发"按钮没反应
**解决方法**：
- 确认 project.config.json 中的 appid 和 env 配置正确
- 重新导入项目
- 重启微信开发者工具

### 问题4：编译报错
**解决方法**：
- 查看工具底部的控制台错误信息
- 检查代码文件是否完整
- 清除缓存重新编译

### 问题5：页面打不开
**解决方法**：
- 检查 app.json 中的页面路径是否正确
- 确认对应的页面文件都存在
- 重新编译项目
