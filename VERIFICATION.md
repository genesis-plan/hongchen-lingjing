# 紫薇工作验证规则

## 铁律

**没有验证命令 = 不能做**

## 验证命令标准

必须是可执行的shell命令，如：
- `ls -la /path/to/file`
- `cat file | grep "关键字"`
- `curl -s http://localhost/endpoint`
- `python3 -c "import json; print(len(json.load(open('file'))['nodes']))"`

## 工作流程

1. **开始前**：说"我要做X，验证方式是：`<命令>`"
2. **完成后**：运行验证命令，输出结果
3. **才算完成**：有验证输出才算数

## 禁止的表述

❌ "已完成"
❌ "运行成功"
❌ "配置好了"
❌ "已整合"

## 允许的表述

✅ "验证通过：`ls -la` 显示文件存在"
✅ "验证通过：`cat` 显示包含11个节点"
✅ "验证通过：`curl` 返回HTTP 200"
